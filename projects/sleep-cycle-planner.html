Overview

This project is a practical Flutter sleep-planning app rather than a starter shell. It calculates recommended sleep and wake times using 90-minute cycles, then layers authentication, onboarding, reminders, profile settings, localization, and local persistence on top of that core flow.

The app is implemented as a mobile-first product with platform targets for Android, iOS, macOS, Linux, Windows, and web. In practice, the strongest implementation is the mobile experience with responsive adaptation where needed.

Architecture is simple and practical: bootstrap in `main.dart`, global app/session state in `app.dart`, go_router-based gating, feature folders under `lib/features/`, and focused storage services instead of heavier repository or DI layers.

Core Features

A focused sleep app with real product wiring behind the UI.

Sleep-cycle suggestions

The dashboard calculates bedtime and wake-up suggestions using 15 minutes to fall asleep plus 90-minute cycles.

Firebase auth flow

Email sign-up, login, password reset, and session restore are wired into the real app flow.

Reminders and notifications

FCM listeners, timezone-aware local notifications, and reminder scheduling are implemented.

Profile metrics

The profile surface derives planned nights, consistency, and average sleep hours from local history.

Localization

English and Kurdish are supported through a custom localization layer with RTL fallbacks.

Local persistence

SharedPreferences stores auth session cache, locale, preferences, reminders, and plan history.

Architecture

Feature-foldered structure with go_router shell navigation, local services, and screen-level state concentrated in large widgets.

Quality status

Tests and analyze are clean, with unit coverage for cycle logic and storage, plus a widget boot smoke test.

Known gaps

README and identifiers are generic, avatar editing is still placeholder, and history/analytics are local-only for now.
